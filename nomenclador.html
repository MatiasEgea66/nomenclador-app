ℹ️ ¿Cómo funciona?

1. La primera vez, creás un nomenclador nuevo e importás tus precios actuales.

2. Cada vez que querés actualizar, subís el archivo maestro (el que descargaste la vez anterior) y la nueva lista de precios.

3. El sistema genera un nuevo archivo maestro con precios actualizados e historial de versiones.

🔄Equivalencias de Códigos
Cuando el archivo de actualización usa un código diferente al tuyo, cargá la equivalencia acá. Por ejemplo: tu código 10 = su código 1.
💾 Las equivalencias se guardan automáticamente en este navegador y se aplican a todas las actualizaciones. No se pierden al cerrar la página.
💡 El multiplicador se aplica al precio obtenido de la actualización. Dejalo vacío o en 1 si no aplica.

⚙️ Configuración de Supabase

Conectá tu base de datos Supabase para sincronizar los datos en la nube y acceder desde cualquier dispositivo.

Paso 1 — Creá el proyecto en Supabase
Entrá a supabase.com → New Project → completá nombre y contraseña.

Paso 2 — Creá la tabla
En tu proyecto: SQL Editor → New query → pegá este código y ejecutalo:
CREATE TABLE nomenclador_data ( key TEXT PRIMARY KEY, value JSONB NOT NULL, updated_at TIMESTAMPTZ DEFAULT NOW() ); ALTER TABLE nomenclador_data ENABLE ROW LEVEL SECURITY; CREATE POLICY "allow_all" ON nomenclador_data FOR ALL USING (true) WITH CHECK (true);
Paso 3 — Copiá las credenciales
En tu proyecto: Project Settings → API → copiá la Project URL y la anon public key.
